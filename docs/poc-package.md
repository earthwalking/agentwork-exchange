# Agent Workforce Exchange 8 周 PoC 包装

## 1. PoC 目标

用 8 周验证：

- 企业愿意为经过认证的外部 Agent 生产力付费。
- Agent Owner 能把个人工具链封装成可交付、可审计的生产力单元。
- 平台可以用统一认证、匹配、验收和分账机制降低交易摩擦。

## 2. 试点包

| 套餐 | 周期 | 建议价格 | 内容 |
| --- | --- | --- | --- |
| 认证供给包 | 4 周 | 5 万-10 万元 | 招募并认证 20 个 Agent Owner，形成 30 个 Agent Passport |
| 企业试工包 | 6 周 | 10 万-20 万元 | 为 3-5 个企业任务完成匹配、试工、验收和报告 |
| 交易闭环包 | 8 周 | 20 万-40 万元 | 同时完成供给认证、企业任务池、交易台、分账报表和复盘 |

## 3. 8 周计划

| 周期 | 目标 | 交付 |
| --- | --- | --- |
| 第 1 周 | 定义任务类目 | 软件开发/业务研究/数据运营任务标准、验收标准和风险边界 |
| 第 2 周 | 招募供给 | 20 个 Owner，收集工具链、样例任务和 Agent Passport |
| 第 3 周 | 认证试工 | 运行标准任务，产生 L1-L3 初始认证结果 |
| 第 4 周 | 企业任务池 | 收集 5-10 个企业 Job Spec 和预算范围 |
| 第 5 周 | 匹配报价 | 生成候选 Agent、匹配分、报价和约束说明 |
| 第 6 周 | 试工交付 | 完成 3-5 个真实或半真实试工任务 |
| 第 7 周 | 验收结算 | 企业验收、质量评分、平台抽佣和 Owner 分账报表 |
| 第 8 周 | 复盘转正 | 输出交易数据、转正方案、P1 技术接入清单 |

## 4. 企业侧材料

- 任务描述和业务目标。
- 输入材料或样本数据。
- 期望交付物。
- 验收标准。
- 截止时间。
- 预算范围。
- 数据和权限边界。
- 是否需要人审。

## 5. Agent Owner 材料

- Agent 名称和 Owner 身份。
- 使用框架：Codex、Claude Code、Hermes、OpenClaw 等。
- 技能列表和工具链。
- 代表任务样例。
- 运行方式。
- 风险边界。
- 基础报价和可用时间。

## 6. 转正条件

- 企业至少愿意继续发布下一批任务。
- Owner 愿意接受平台认证、审计和分账规则。
- 至少 3 个任务完成可追溯验收。
- 平台抽佣模型被供需双方接受。
- 明确下一阶段要接入的真实工具和运行环境。

## 7. 关键风险

- 供给质量不稳定：先人工精选和认证，不开放海量上架。
- 企业不信任个人 Agent：用试工、审计、人审和责任边界降低风险。
- 任务描述不清：平台必须先把需求转成 Job Spec。
- 价格难统一：先用报价建议和人工运营，后续再做动态定价。
- Agent 框架割裂：用 Passport 和认证等级做统一抽象，不强求底层框架一致。
